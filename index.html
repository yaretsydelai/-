从标注员到「规则的定义者」——一名数据工程师在 Agent 浪潮中的一年

过去一年，我在两家公司、三个项目里完成了从「执行规则」到「定义规则」的转变。从科大讯飞录音笔 Pokee 的 Function Call 能力建设，到晓多电商 Agent 的评测体系搭建——这是我用 Python、Prompt 工程和标注管理，把模糊需求转成可度量成果的实录。

Agent 时代数据工程最稀缺的，不是工具，而是一种愿意把自己放到规则、数据、执行、工程每一个环节去追根究底的耐心。当我面对「多轮开麦失效」这样的 Badcase 时，单方面修规则永远不够——真正的闭环，是从 AI 输出、训练数据、模型评测到线上业务四层都验证过一遍。

我是一名把「数据 + Prompt + 规则」当成工程问题来拆解的人。

计算机专业科班出身，过去一年里扎在 AI Agent 一线——从科大讯飞 Pokee 的 Function Call 数据建设，到晓多电商客服 Agent 的评测体系搭建。我习惯把模糊的业务需求拆成可度量的维度，用 Python 脚本把重复劳动批量化，用 Prompt 工程把规则对齐，用标注闭环把质量稳住。

我熟练使用 Python 做数据清洗、批处理和分析脚本开发；精通 Prompt Engineering，擅长为多场景设计和优化 Prompt；具备从数据构建到模型上线的 AI 项目全流程管理经验。比起「把任务做完」，我更在乎「从规则、数据、执行、工程四个层面把问题真正闭环」。

业余时间在研究 Agent 自主部署与自动化工作流，也在把这些经验整理成可复用的方法论——这些会陆续出现在札记里。
