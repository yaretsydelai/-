01

背景  — the brief

What was the problem?

中小电商商家面临一个长期的三难：客服人力成本高、响应慢、培训周期长。一个新客服从上岗到独当一面往往需要 2-3 周，而双十一这样的流量高峰根本等不及。

项目的目标是构建一个能够覆盖售前—售中—售后全链路的智能客服 Agent——让常见咨询被自动处理，让复杂问题能无缝转人工。核心难点不在"做一个客服机器人"，而在"让它真正可用"。

作为数据运营负责人，我要把业务方的模糊需求转成可度量的数据集、可验证的评测标准，并在有限的团队产能下交付足够多的高质量数据。

02

我的动作  — what I did

Three stages, one loop

我把工作拆成三个阶段：需求承接 → 评测全流程管理 → 结果交付与归因评估，三个阶段在项目中是一个持续循环的闭环。

  1. 需求承接。与业务方明确项目背景、交付周期、评测数据来源与实现结果，对评测数据集进行体系化构架，组织团队对原始数据做分类、筛选和试标，建立初版评测集文档。
  2. 自研脚本提效。自研 Python 自动化数据构建脚本——场景矩阵生成 + Prompt 模板化 + Qwen API 批量调用，使单人日产能 20 → 40。
  3. 评测维度制定。依据业务方需求与使用场景，制定 5 个量化评测维度与权重分配。
  4. 多维归因体系。对所有 Badcase 整理分类，建立情绪处理分级、工具失败四类兜底、参数幻觉、多意图漏处理等多维度归因体系。
  5. 可视化评估报告。使用 Python 对评测数据可视化分析，直观呈现 Agent 在不同维度下的优劣分布，生成评估报告，配合业务方提出风险点与改进方向。

Biggest wins

在单个维度上，工具调用准确性 +34 个百分点、回复人性化 +29 个百分点——这两个维度恰好是智能客服最难做、最容易出 Badcase 的地方。

这个项目让我第一次完整地走通了"业务需求 → 数据构建 → 评测分析 → 优化闭环"的链路。它塑造了我今天做事的基本方法：拆解维度、量化指标、可视化反馈。
