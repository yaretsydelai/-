一 · The Self

一名把数据
当成手艺的人

On being a data engineer.

我是张嘉烜，23 岁，计算机科班出身。过去一年我全身心扎在 AI Agent 的一线——做数据、写规则、管团队，也亲手跑通从 0 到 1 的项目。表面上我的 title 是"数据工程师"，但更准确地说，我是那个在规则、数据、执行、工程四个维度之间跑来跑去，把模糊需求一点点啃成可度量成果的人。

刚入行时我是一名"执行规则的训练师"——别人告诉我怎么标，我照做。但真正让我成长的，是一次次面对那些规则解释不清的 Badcase：线上多轮开麦失效、工具调用参数幻觉、多意图漏处理……每一个都逼着我从"照做"走向"去追问为什么"。

后来我慢慢意识到：AI 数据工程最稀缺的，不是工具，而是一种愿意把自己放到每一个环节去追根究底的耐心。

现在的我，能独立输出规则文档、主导跨团队联调、组织 15 人标注团队、自研 Python 脚本把单人日产能翻倍、从 0 到 1 建立评测体系。但我更在意的是——我学会了怎么在"按期交付"和"质量闭环"之间找平衡，学会了怎么把一个 Badcase 从 AI 输出一路回溯到规则盲区。

二 · Working Principles

我相信的四件事

The way I work
01

规则必须写得让一个新人也能照做。

规则文档不是写给自己看的。爬坡期每周三次跨团队联调会，逼着我把每个维度都讲清楚。不能自证的规则，就是盲区。

02

Badcase 要从四层闭环。

规则、数据、执行、工程——任何一层没修都算没修完。最后还要从 AI 输出、训练数据、模型评测、线上业务四层分别验证，确认问题真正闭环。

03

能脚本化的事情不靠加班。

自研 Python 场景矩阵 + Prompt 模板 + Qwen API 批量调用，单人日产能 20 → 40。重复劳动应该被批量化、被自动化，不应该被熬夜解决。

04

质量是设计出来的，不是抽检出来的。

3-5 天准入考核 + 9 步标注闭环 + 质检漏斗。标注准入率 95% 不是抽查出来的，是在流程里一层层筛出来的。

四 · Off Hours

工作之外，
我在做些什么

Between projects.

业余时间我喜欢折腾 Agent 框架——自己搭过 Open Claw 和 Hermes Agent，跑过各种工作流试验，也在尝试把 Claude Code 接到日常标注 + 评测流程里，做一些能立刻减负的小工具。这些尝试回头都成了我工作里的"第二条腿"。

我也喜欢把做过的事情整理成可复用的方法论。一次 Badcase 归因、一次评测维度设计、一次规则文档迭代——这些都值得被写下来。所以你在这个网站看到的"札记"，就是我正在准备要发布的东西。

除此之外，我相信快速学习是 AI 行业从业者最基本的素养。所以我保持着一种"每周读一点、每月做一个小实验"的节奏——技术迭代快，但只要保持脚步，就不会掉队。
